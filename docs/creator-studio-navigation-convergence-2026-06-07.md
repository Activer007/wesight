# Creator Studio Navigation Convergence - Phase 13C Check

## Navigation

Creator Studio top-level navigation is now reduced to four default entries:

- Start
- Inspiration
- Builder
- Assets

Gallery, Templates, and Nano Library remain available as segmented views inside Inspiration. Board, Batch, and Image Tools remain implemented but are no longer exposed as equal top-level tabs.

## Inspiration

Inspiration keeps the existing capability surface while lowering entry choice cost:

- Gallery search, filters, case detail, and "use case" still work inside Inspiration.
- Templates stay searchable through the template library section and still open Builder with template fields.
- Nano Prompt Library keeps its own search, filters, sync status, and prompt detail drawer.
- Any case, template, or Nano prompt has a clear path into Builder.

Nano prompt details now follow the one-primary-action rule:

- Primary action: Use in Builder.
- Secondary actions: save Recipe, save Prompt Asset, add to Board, send to Cowork, create Batch, and open source are grouped under More actions.

## Builder

Builder defaults to a short first-run surface:

- Source summary
- One-line brief autofill
- Core inputs: topic, platform, aspect ratio, required text, visual style
- Materials
- Prompt quality summary
- Final prompt preview
- Primary send-to-Cowork action

Advanced mode keeps the existing power-user surface:

- Project switch and creation
- Recipe import, reuse, export, and scheduling
- Production Package metrics and governance
- Board, Batch, and Image Tools entrypoints
- Template fields and template method notes
- Full PromptSpec parameters
- Full lint issue details
- PromptSpec preview, copy, Prompt Asset, Recipe, and Seedream generation actions
- Context Pack, Creative Directions, and recommended runtime details

## Notes

- No Creator Studio production asset model changes were introduced.
- No Board, Batch, Image Tools, Gallery, Templates, Nano, Recipe, PromptSpec, or Production Package capabilities were removed.
- The change only reduces default exposure and moves advanced workflows behind contextual or advanced entrypoints.
